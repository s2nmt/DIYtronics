Giai Đoạn 1: Triển Khai - Làm Rõ Yêu Cầu

Ví dụ: Khách hàng yêu cầu 1 website và ứng dụng hiển thị nhiệt độ, độ ẩm. Đây là các yêu cầu khách hàng đưa ra:

STT Yêu Cầu
1 Giám sát và điều khiển nhiệt độ độ ẩm ở app và website
2 Cảnh báo tối đa 10 alarm qua app, lưu cảnh báo và giá trị theo thời gian thực, có thể xem trên app và website
3 Mỗi thiết bị được quản lý id riêng ứng với user riêng
4 Khai báo user và add id esp
5 Lưu dữ liệu 3 tháng trên server. Lưu theo cảnh báo và timeout setup
6 1 user chính giám sát 1 device A, vài user phụ có thể cùng giám sát
7 Chức năng gửi mail khi có alarm (có thể set alarm, nội dung) và set tối đa 5 email, trên 1 user
8 Hỗ trợ kết nối MQTT cho wifi và sim

Đánh giá:

Dự án có thể làm được, chưa có vấn đề phức tạp.

Giai đoạn 2: Phân Tích Chi Tiết Theo Từng Phần

Yêu Cầu 1: Giám sát và điều khiển nhiệt độ độ ẩm ở app và website

Hạng Mục Câu Hỏi Cần Làm Rõ Mô Tả Báo Giá
Đăng nhập/Đăng ký APP và website cần đăng ký đăng nhập tài khoản không? Frontend: Cần có 1 màn hình đăng nhập bao gồm tài khoản, mật khẩu, quên mật khẩu, đăng kí.
Backend: API đăng nhập, đăng ký, xác thực token 		✓
Có cần lấy OTP không? phương thức từ email hay số điện thoại ,..? Frontend: Màn hình nhập OTP, đếm ngược thời gian
Backend: Triển khai OTP qua Zalo, lưu trữ và xác thực OTP 		✓
Giao diện chính Hiển thị giá trị nhiệt độ độ ẩm (số lượng giá trị cần hiển thị, cố định hay linh hoạt?) Frontend: Thiết kế màn hình hiển thị dữ liệu nhiệt độ và độ ẩm theo thời gian thực
Backend: API lấy dữ liệu real-time, WebSocket hoặc polling 		✓
Hiển thị biểu đồ (số lượng biểu đồ, biểu đồ dạng nào, bao nhiêu giá trị?) Frontend: Phát triển biểu đồ hiển thị dữ liệu lịch sử với các tùy chọn loại biểu đồ
Backend: API lấy dữ liệu lịch sử theo khoảng thời gian 		✓
Điều khiển Điều khiển đèn (hỏi bao nhiêu đèn, cố định hay thêm sửa xóa, phương pháp thêm đèn ntn?) Frontend: 1 màn hình có thể điều khiển on/off, thêm sửa xóa đèn
Backend: API điều khiển thiết bị, quản lý danh sách đèn 		✓
Cảnh báo Hiển thị trạng thái cảnh báo, hiển thị như thế nào? Trong app hay khi tắt app vẫn? Frontend: Thiết kế hệ thống cảnh báo với thông báo push notification, và thông báo trong giao diện home
Backend: Xử lý logic cảnh báo, gửi push notification, lưu trữ lịch sử cảnh báo 		✓

Yêu Cầu 2: Cảnh báo tối đa 10 alarm qua app, lưu cảnh báo và giá trị theo thời gian thực

Hạng Mục Câu Hỏi Cần Làm Rõ Mô Tả Báo Giá
Xuất file Excel Định dạng file Excel như thế nào? Có cần filter, sort không? Frontend: Thiết kế 1 màn hình có thể xuất dữ liệu lịch sử ra file Excel với đầy đủ thông tin và có thể filter
Backend: API xuất file Excel, xử lý dữ liệu và format Excel 		✓
Lịch sử cảnh báo Hiển thị bao nhiêu bản ghi? Có phân trang không? Frontend: Thiết kế 1 màn hình hiển thị lịch sử cảnh báo với tìm kiếm và lọc
Backend: API lấy lịch sử cảnh báo với phân trang, filter, search 		✓

Yêu Cầu 3: Mỗi thiết bị được quản lý id riêng ứng với user riêng

Hạng Mục Câu Hỏi Cần Làm Rõ Mô Tả Báo Giá
Khai báo user và add ESP ID Cần API nào để quản lý user và device? Có cần phân quyền không? Frontend: Màn hình quản lý user, màn hình thêm/sửa/xóa device
Backend: Phát triển hệ thống backend quản lý user và device với authentication và authorization 		✓

Yêu Cầu 4: Khai báo user và add id esp

Hạng Mục Câu Hỏi Cần Làm Rõ Mô Tả Báo Giá
Thêm thiết bị Quét QR hay nhập ID thiết bị. Thiết bị gồm những thông tin gì? Frontend: Màn hình quét QR code, màn hình nhập thủ công thông tin thiết bị
Backend: API thêm thiết bị mới, validate thông tin thiết bị 		✓
Kết nối WiFi Theo phương pháp nào, gồm phần bluetooth, truyền nhận data bao gồm ssid password? Frontend: Màn hình cấu hình WiFi, nhập SSID và password
Backend: API gửi cấu hình WiFi đến thiết bị qua Bluetooth hoặc WiFi AP mode 		✓
Có cần check thiết bị kết nối thành công không? Frontend: Hiển thị trạng thái kết nối, thông báo kết quả
Backend: API kiểm tra trạng thái kết nối thiết bị 		✓
Có WiFi static không? Frontend: Màn hình cấu hình IP tĩnh (IP, Subnet, Gateway, DNS)
Backend: API lưu và gửi cấu hình IP tĩnh đến thiết bị 		✓

Yêu Cầu 5: Lưu dữ liệu 3 tháng trên server. Lưu theo cảnh báo và timeout setup

Hạng Mục Câu Hỏi Cần Làm Rõ Mô Tả Báo Giá
Timeout setup Timeout cho từng loại dữ liệu là bao nhiêu? Có thể tùy chỉnh không? Frontend: Thiết kế màn hình cấu hình timeout cho việc lưu trữ dữ liệu và cảnh báo
Backend: API lưu cấu hình timeout, xử lý logic timeout cho từng loại dữ liệu 		✓

Yêu Cầu 6: 1 user chính giám sát 1 device A, vài user phụ có thể cùng giám sát

Hạng Mục Câu Hỏi Cần Làm Rõ Mô Tả Báo Giá
Chia sẻ thiết bị 1 màn hình chia sẻ thiết bị cho user phụ. Cách share như thế nào? Ví dụ share qua email hoặc QR Frontend: Màn hình chia sẻ thiết bị, hiển thị QR code, nhập email để share
Backend: API tạo link/QR code chia sẻ, gửi email mời, quản lý quyền truy cập user phụ 		✓

Yêu Cầu 7: Chức năng gửi mail khi có alarm (có thể set alarm, nội dung) và set tối đa 5 email, trên 1 user

Hạng Mục Câu Hỏi Cần Làm Rõ Mô Tả Báo Giá
Màn hình set email Có thể tùy chỉnh nội dung email không? Template email như thế nào? Frontend: Thiết kế màn hình quản lý danh sách email nhận cảnh báo với tối đa 5 email, tùy chỉnh nội dung email
Backend: API lưu danh sách email, lưu template email 		✓
Server gửi email Dùng email service nào? SMTP server riêng hay third-party? Frontend: -
Backend: Phát triển hệ thống gửi email tự động từ server khi có cảnh báo, tích hợp SMTP hoặc email service 		✓

Yêu Cầu 8: Hỗ trợ kết nối MQTT cho wifi và sim

Hạng Mục Câu Hỏi Cần Làm Rõ Mô Tả Báo Giá
Firmware MQTT broker nào? QoS level? Có cần retain message không? Frontend: -
Backend/Firmware: Phát triển firmware ESP32 hỗ trợ MQTT qua WiFi và SIM với các tính năng: connect, disconnect, publish, subscribe, reconnect, QoS handling 		✓
