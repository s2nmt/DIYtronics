    ESP32 OTA Firmware Update over HTTPS

    Updating ESP32 firmware over the air (OTA) lets you push new versions to devices without USB or physical access. This post walks through a simple OTA flow: the device checks a secure HTTPS URL for a new firmware binary, downloads it if the server says an update is available, writes it with the Arduino Update library, and reboots. We use a Root CA certificate so the ESP32 can verify your server over TLS.

    What is OTA (Over-The-Air) Update?

    OTA update means sending new firmware to a device over the network (Wi‑Fi, Ethernet, or cellular) instead of flashing via USB or a programmer. For ESP32, the device downloads a .bin file, writes it to the next app partition (or a dedicated OTA partition), and restarts to run the new code. That’s ideal for deployed or hard-to-reach devices.

    Overview of the Code

    The sketch does the following:

    • Connects to Wi‑Fi using your SSID and password.
    • Uses WiFiClientSecure with a Root CA certificate so the HTTPS connection is verified.
    • Sends a GET request to a firmware URL and passes the current version in a custom header X-Current-Version.
    • If the server returns 200 OK, treats the response body as the new firmware: streams it into Update, then reboots on success.
    • If the server returns 304 Not Modified, considers the device up to date and skips the update.
    • Runs the update check periodically in loop() (e.g. every 2 minutes in the example).

    Full Sketch

    Here is the complete ESP32 sketch for OTA over HTTPS with version checking:

    #include <WiFi.h>
#include <HTTPClient.h>
#include <Update.h>
#include <WiFiClientSecure.h>

const char* ssid = "your wifi";
const char* password = "your password";
const char* firmwareUrl = "https://your_domain/firmware/device_v3.bin";
const char* currentVersion = "1.0.0";

// Root CA certificate for your server (PEM format)
// Extract with: openssl s_client -connect your-server.com:443 -showcerts
const char* rootCACert = R"EOF(
-----BEGIN CERTIFICATE-----
-----END CERTIFICATE-----
)EOF";

void checkForUpdate() {
    WiFiClientSecure client;
    client.setCACert(rootCACert);

    HTTPClient https;
    https.begin(client, firmwareUrl);
    https.addHeader("X-Current-Version", currentVersion);

    int httpCode = https.GET();

    if (httpCode == HTTP_CODE_OK) {
        int contentLength = https.getSize();
        if (contentLength <= 0) {
            Serial.println("Invalid content length");
            return;
        }

        if (!Update.begin(contentLength)) {
            Serial.println("Not enough space for OTA");
            return;
        }

        WiFiClient* stream = https.getStreamPtr();
        size_t written = Update.writeStream(*stream);

        if (written == contentLength) {
            Serial.println("Firmware written successfully");
        } else {
            Serial.printf("Only wrote %d of %d bytes\n", written, contentLength);
        }

        if (Update.end()) {
            if (Update.isFinished()) {
                Serial.println("Update successful. Rebooting...");
                ESP.restart();
            }
        } else {
            Serial.printf("Update error: %s\n", Update.errorString());
        }
    } else if (httpCode == 304) {
        Serial.println("Firmware is up to date");
    } else {
        Serial.printf("HTTP error: %d\n", httpCode);
    }

    https.end();
}

void setup() {
    Serial.begin(115200);
    pinMode(2, OUTPUT);
    WiFi.begin(ssid, password);
    while (WiFi.status() != WL_CONNECTED) {
        delay(500);
    }
    // checkForUpdate();  // optional: check once at boot
}

void loop() {
    static uint32_t lastCheck = 0;
    if (millis() - lastCheck > 120000) {
        checkForUpdate();
        lastCheck = millis();
    }
    digitalWrite(2, HIGH);
    delay(2000);
    digitalWrite(2, LOW);
    delay(2000);
}

    What You Need to Configure

    • ssid / password — Your Wi‑Fi credentials.
    • firmwareUrl — Full HTTPS URL to the firmware binary (e.g. https://your_domain/firmware/device_v3.bin).
    • currentVersion — Version string reported to the server in X-Current-Version. The server can compare this and return 304 when no update is needed.
    • rootCACert — PEM of the CA that signed your server’s certificate. You can get it with: openssl s_client -connect your-server.com:443 -showcerts and copy the “BEGIN CERTIFICATE” block (usually the server cert or the CA that signed it).

    Server-Side Behavior

    Your server should:

    1. Accept GET requests to the firmware URL.
    2. Read the X-Current-Version header.
    3. If that version is already the latest, respond with 304 Not Modified and no body (saves bandwidth and flash wear).
    4. If a newer build exists, respond with 200 OK, Content-Length set to the binary size, and the .bin file as the body.

    That way the ESP32 only downloads and flashes when there is actually a new version.

    Building the Firmware Binary

    In Arduino IDE: Sketch → Export compiled Binary. The .bin is created in the sketch folder. Upload this file to your server at the path used in firmwareUrl. For PlatformIO, the built binary is typically under .pio/build/<env>/firmware.bin; use the same file (or a renamed copy) on the server.

    Security and Best Practices

    • HTTPS and Root CA — Using setCACert(rootCACert) ensures the ESP32 only talks to a server that presents a certificate signed by your CA, which reduces man-in-the-middle risk.
    • Version header — Sending X-Current-Version and having the server return 304 when up to date avoids unnecessary downloads and flash writes.
    • Check interval — The example checks every 2 minutes (120000 ms). For production you might use a longer interval (e.g. 24 hours) or trigger updates via a message (e.g. MQTT) instead of polling.
    • First flash — The first time you flash the device you must use USB/serial; after that, OTA can deliver all future updates.

    With this setup, your ESP32 can securely check for and install firmware updates over HTTPS without physical access.
