This is Part 1 of our Yocto on BeagleBone Black series. We build a minimal Linux image with Yocto Scarthgap (5.0), flash it to a microSD card, and boot the board. The steps below follow the reference article above, adapted into English with the same commands and layer layout.

1. Prepare the build host

Use Ubuntu 22.04 LTS (or equivalent). A multi-core machine with enough disk space is recommended—the reference build on an Intel i7-13700H (14 cores / 20 threads) took a little over one hour.

Install host packages per the Yocto Project Reference Manual (section 1.3.2.1):

sudo apt-get update
sudo apt-get -y install gawk wget git-core diffstat unzip texinfo gcc-multilib \
  build-essential chrpath socat cpio python3 python3-pip python3-pexpect \
  libsdl1.2-dev xterm make xsltproc docbook-utils fop dblatex xmlto lz4

2. Clone source code

We use Poky (the Yocto reference distribution) on branch scarthgap (Yocto 5.0), plus the layers needed for BeagleBone Black:

git clone -b scarthgap git://git.yoctoproject.org/poky.git
cd poky

git clone -b scarthgap git://git.openembedded.org/meta-openembedded
git clone -b scarthgap git://git.yoctoproject.org/meta-ti
git clone -b scarthgap https://git.yoctoproject.org/meta-arm

Directory layout after cloning:

poky/
├── meta/                    # OE-Core
├── meta-poky/
├── meta-yocto-bsp/
├── meta-openembedded/
├── meta-ti/
└── meta-arm/

3. Setup and build

3.1 Create the build environment

From inside poky/, run:

source oe-init-build-env build-bbb

This command:

  • Sets $OEROOT — root of the Yocto / Poky tree
  • Updates $PATH — adds BitBake to your shell session
  • Sets $BUILDDIR — output directory (./build-bbb; use separate build dirs when targeting multiple boards)
  • Sets $BBPATH — so BitBake can find layers
  • Creates the $BUILDDIR folder structure if it does not exist

3.2 Edit configuration files

For simplicity we edit local.conf and bblayers.conf directly under build-bbb/conf/.

Machine — open build-bbb/conf/local.conf and set:

MACHINE ?= "beaglebone"

Layers — open build-bbb/conf/bblayers.conf and add the OpenEmbedded, ARM, and TI layers. Use absolute paths on your machine (example below—replace /home/user/Learning/poky with your path):

POKY_BBLAYERS_CONF_VERSION = "2"

BBPATH = "${TOPDIR}"
BBFILES ?= ""

BBLAYERS ?= " \
  /home/user/Learning/poky/meta \
  /home/user/Learning/poky/meta-poky \
  /home/user/Learning/poky/meta-yocto-bsp \
  /home/user/Learning/poky/meta-openembedded/meta-oe \
  /home/user/Learning/poky/meta-arm/meta-arm-toolchain \
  /home/user/Learning/poky/meta-arm/meta-arm \
  /home/user/Learning/poky/meta-ti/meta-ti-bsp \
  /home/user/Learning/poky/meta-ti/meta-ti-extras \
  /home/user/Learning/poky/meta-ti/meta-beagle \
  "

3.3 Build the image

Build the smallest reference image—it boots a minimal OS with very few packages, so the first build is faster than a desktop image:

bitbake core-image-minimal

Expect the first build to take roughly one hour or more depending on CPU, disk, and network. Subsequent builds are much faster when the download and shared-state caches are warm.

4. Flash the image to microSD

4.1 Find the output image

If you are unsure where BitBake placed the artifacts, query the recipe environment:

bitbake -e core-image-minimal | grep ^WORKDIR=

bitbake -e <recipe> prints all variables set for that recipe. grep ^WORKDIR= filters the work directory line. Example output:

WORKDIR="/home/user/Learning/poky/build-bbb/tmp/work/beaglebone-oe-linux-gnueabi/core-image-minimal/1.0-r0"

Then open the deploy folder inside that path:

cd /home/user/Learning/poky/build-bbb/tmp/work/beaglebone-oe-linux-gnueabi/core-image-minimal/1.0-r0
cd deploy-core-image-minimal-image-complete

You can also find images under build-bbb/tmp/deploy/images/beaglebone/.

4.2 Write the image to the SD card

Use the .wic.xz image—it is compressed and works well with bmaptool for faster flashing.

Insert the microSD card and check the device name:

lsblk

If partitions are auto-mounted, unmount them first (adjust paths to match your system):

sudo umount /media/user/boot /media/user/root

Flash with bmaptool (replace /dev/mmcblk0 with your card device—double-check to avoid wiping the wrong disk):

sudo bmaptool copy core-image-minimal-beaglebone.wic.xz /dev/mmcblk0

5. Boot BeagleBone Black

Insert the microSD card into the BBB and power on.

  • Hold the S2 (boot) button while powering on to boot from MMC1 (microSD). Without S2, the board boots from MMC2 (onboard eMMC).
  • Log in as user root with no password on the default minimal image.
