MQTT (Message Queuing Telemetry Transport) is a lightweight messaging protocol designed for IoT devices. Setting up an MQTT broker on Ubuntu allows you to create a central hub for your IoT projects.

What is MQTT Broker?

An MQTT broker is a server that receives messages from clients (publishers) and routes them to other clients (subscribers) based on topics. It acts as a central communication hub for IoT devices, enabling efficient and reliable message exchange with minimal overhead.

Installing Aedes MQTT Broker

First, update your package list and install nodejs 18.x:

sudo apt update
curl -fsSL https://deb.nodesource.com/setup_18.x | sudo -E bash -

After installation, we are install nodejs to run aedes broker:

sudo apt install nodejs -y

Now, create a new directory for your MQTT broker project and install the required packages:

mkdir mqtt-broker
cd mqtt-broker
npm install aedes ws http net

certbot certonly --standalone -d mqtt.example.com

to get cert access this link https://letsencrypt.org/certs/isrgrootx1.pem

Step 1: Code example to run aedes broker

Create a file named broker.js and add the following code:

const aedes = require('aedes')()
const tls = require('tls')
const fs = require('fs')
const https = require('https')
const ws = require('ws')

// ===== Load cert =====
const options = {
    key: fs.readFileSync('/etc/letsencrypt/live/mqtt.example.com/fullchain.pem'),
    cert: fs.readFileSync('/etc/letsencrypt/live/mqtt.example.com/privkey.pem')
}

// ===== MQTTs (TLS) =====
tls.createServer(options, aedes.handle).listen(8883, () => {
    console.log('MQTTS broker running on port 8883')
})

// ===== WSS (WebSocket Secure) =====
const httpsServer = https.createServer(options)
const wsServer = new ws.Server({ server: httpsServer })

wsServer.on('connection', (socket) => {
    const stream = ws.createWebSocketStream(socket)
    aedes.handle(stream)
})

httpsServer.listen(8084, () => {
    console.log('MQTT WSS running on port 8084')
})

// ===== Logs =====
aedes.on('client', (client) => {
    console.log('Client connected:', client.id)
})

aedes.on('publish', (packet, client) => {
    if (client) {
    console.log(`[${client.id}] ${packet.topic}: ${packet.payload.toString()}`)
    }
})

Step 2: Run aedes broker

Run the broker with the following command:

node broker.js

Output:

MQTTS running on  port 8883
MQTT WSS running on port 8084

Testing the MQTT Broker

After configuring, you can use the following code example to test the broker. File cert.pem you have to download from server.

Python code port 8084:

import ssl
import certifi
import paho.mqtt.client as mqtt

BROKER = "your IP"
PORT = 8084
TOPIC = "test/topic"

def on_connect(client, userdata, flags, rc):
    print("Connected:", rc)
    client.subscribe(TOPIC)
    client.publish(TOPIC, "Hello from Python WSS")

def on_message(client, userdata, msg):
    print(msg.topic, msg.payload.decode())

client = mqtt.Client(transport="websockets")
client.on_connect = on_connect
client.on_message = on_message

client.tls_set(
    ca_certs=certifi.where(),
    tls_version=ssl.PROTOCOL_TLS_CLIENT
)

client.connect(BROKER, PORT, 60)
client.loop_forever()
