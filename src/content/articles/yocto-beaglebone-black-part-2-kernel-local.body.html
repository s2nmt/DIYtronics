This is Part 2 of the Yocto on BeagleBone Black series. After Part 1 (build environment and a working SD image), we use BitBake to download and unpack the kernel so the full source tree is available locally on your PC.

Prerequisites

Complete Part 1 first. You should already have Poky Scarthgap cloned, build-bbb configured with MACHINE ?= "beaglebone", and a successful bitbake core-image-minimal run.

Re-enter the build environment:

cd poky
source oe-init-build-env build-bbb

Which kernel recipe?

Target the virtual provider rather than hard-coding a recipe name. Confirm on your setup:

bitbake -e virtual/kernel | grep ^P=
bitbake -e virtual/kernel | grep ^PV=

Example output:

P="linux-ti-staging"
PV="6.6.xx"

Use virtual/kernel in the commands below so they stay valid even if the underlying recipe name changes.

Fetch and unpack the kernel locally

bitbake -c fetch virtual/kernel
bitbake -c unpack virtual/kernel

Downloads go to build-bbb/downloads/. The unpacked tree lands under build-bbb/tmp/work/.

Find the local kernel source path

bitbake -e virtual/kernel | grep ^S=

Typical path pattern:

S=".../build-bbb/tmp/work/beaglebone-oe-linux-gnueabi/linux-ti-staging/<version>/git"

Go there and verify:

cd $(bitbake -e virtual/kernel | sed -n 's/^S="\([^"]*\)".*/\1/p')
pwd
ls
# expect: arch/ drivers/ include/ kernel/ Makefile  ...

Build the kernel only (optional)

bitbake virtual/kernel

Artifacts land under build-bbb/tmp/deploy/images/beaglebone/ (e.g. zImage, device-tree blobs).

Keep a persistent local copy with devtool

Files under tmp/work can be wiped on rebuild. For ongoing changes, use the recipe name from bitbake -e virtual/kernel | grep ^P=:

devtool modify linux-ti-staging
bitbake virtual/kernel

Sources are copied to build-bbb/workspace/sources/. When done, save your work into a layer:

devtool finish linux-ti-staging <your-meta-layer>

Or reset:

devtool reset linux-ti-staging

Useful paths (summary)

  • Downloads: build-bbb/downloads/
  • Unpacked work copy: build-bbb/tmp/work/.../<kernel-recipe>/.../git/
  • devtool workspace: build-bbb/workspace/sources/
  • Built kernel output: build-bbb/tmp/deploy/images/beaglebone/

Summary

bitbake -c fetch and bitbake -c unpack virtual/kernel place the BeagleBone kernel source on your host. Use bitbake -e virtual/kernel to print the exact path, and devtool modify when you need a lasting local tree for patches and driver work.
