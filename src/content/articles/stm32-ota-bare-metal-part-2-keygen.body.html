Series: STM32F103 Bare-Metal Secure OTA Bootloader — Part 1: Overview & Architecture.

Source: bootloader/tools/keygen.py in STM32F103C8T6-ota-base-metal.

This post walks through keygen.py line by line. The script is the trust anchor setup for the OTA stack: it derives an ECDSA P-256 key pair from a fixed seed, then writes the public key only into C source so the bootloader can verify signed images. The private key stays on the PC and is reused by sign_image.py and main.py --sign.

1. Where it fits

keygen.py
    └── writes bootloader/lib/boot/src/boot_pubkey.c  (64-byte public key)

sign_image.py / main.py --sign
    └── import derive_signing_key() from keygen.py
    └── sign payload with private key

bootloader verify.c
    └── ECDSA verify using bootloader_pubkey[64]

After running keygen.py you must rebuild and reflash the bootloader. Images signed before a key change will no longer verify.

2. Dependencies and constants

Full script (~45 lines):

#!/usr/bin/env python3
"""Generate bootloader_pubkey.c from deterministic dev fixture key."""

from __future__ import annotations

import hashlib
import sys
from pathlib import Path

KEY_SEED = b"stm32-bare-metal-dev-fixture"
OUT_C = Path(__file__).resolve().parent.parent / "lib" / "boot" / "src" / "boot_pubkey.c"
  • KEY_SEED — fixed byte string used to derive the private key. Same seed always yields the same key pair (handy for tutorials and CI). Not for production — anyone who knows the seed can sign firmware.
  • OUT_C — output path resolved relative to the script: bootloader/lib/boot/src/boot_pubkey.c. Parent of tools/ is bootloader/, then lib/boot/src/.
  • ecdsa — Python package (pip install ecdsa). Uses curve NIST256p (same as ECDSA P-256 / secp256r1 on the MCU).

If ecdsa is missing, the script prints Thieu ecdsa. Chay: pip install ecdsa and exits with code 1.

3. derive_signing_key()

def derive_signing_key() -> SigningKey:
    priv = int.from_bytes(hashlib.sha256(KEY_SEED).digest(), "big") % NIST256p.order
    if priv == 0:
        priv = 1
    return SigningKey.from_secret_exponent(priv, curve=NIST256p)
  1. SHA-256 the seed — hashlib.sha256(KEY_SEED) produces 32 bytes. This is a simple key derivation: seed → digest → scalar. Production systems would use proper KDF (e.g. HKDF) and a secret stored in an HSM.
  2. Interpret as integer — int.from_bytes(..., "big") treats the digest as an unsigned big-endian integer.
  3. Reduce modulo curve order — % NIST256p.order ensures the scalar lies in the valid range for secp256r1 (order ≈ 2256). Without this step, the value could be out of range and key creation would fail.
  4. Reject zero — private key 0 is invalid; force to 1 if the modulo result is 0 (extremely unlikely with this seed).
  5. Build SigningKey — SigningKey.from_secret_exponent(priv, curve=NIST256p) creates the object used later to sign (sign_image.py) or to extract the public key (main()).

This function is imported by sign_image.py so signing and verification always use the same deterministic dev key — no separate key file on disk.

4. main() — emit boot_pubkey.c

def main() -> int:
    sk = derive_signing_key()
    pub = sk.get_verifying_key().to_string()

    lines = ["/* Auto-generated by tools/keygen.py — do not edit */", "#include <boot_pubkey.h>", ""]
    lines.append("const uint8_t bootloader_pubkey[64] = {")
    for i in range(0, 64, 8):
        chunk = ", ".join(f"0x{b:02X}" for b in pub[i : i + 8])
        lines.append(f"    {chunk},")
    lines.append("};")
    lines.append("")

    OUT_C.write_text("\n".join(lines), encoding="utf-8")
    print(f"Wrote {OUT_C}")
    return 0
  • get_verifying_key().to_string() — returns 64 raw bytes: 32-byte X coordinate + 32-byte Y coordinate (uncompressed public point, no 0x04 prefix). This matches what lib/crypto expects in bootloader_pubkey[64].
  • C array formatting — loops in steps of 8 bytes per line for readability. Each byte is 0xNN uppercase hex.
  • Overwrite boot_pubkey.c — the file is regenerated entirely; manual edits are lost (header comment warns this).

Example output (first lines):

/* Auto-generated by tools/keygen.py — do not edit */
#include <boot_pubkey.h>

const uint8_t bootloader_pubkey[64] = {
    0x9E, 0xDF, 0x13, 0xD9, 0x7B, 0xB6, 0xC6, 0xE6,
    ...
};

Header boot_pubkey.h only declares extern const uint8_t bootloader_pubkey[64]; — verify code includes this and passes the array to micro-ecc ECDSA verify.

5. Link to sign_image.py

sign_image.py imports the same helper:

from keygen import derive_signing_key

def build_header(payload: bytes, version: int) -> bytes:
    digest = sha256(payload)
    sk = derive_signing_key()
    signature = sk.sign_digest(digest, sigencode=...)

Flow: hash the app payload with SHA-256 → sign the 32-byte digest with the derived private key → pack into the 256-byte IMG1 header. The bootloader recomputes SHA-256 and verifies the 64-byte (r, s) signature against bootloader_pubkey. If keygen.py was never run (or bootloader not reflashed after keygen), verify fails and the image is rejected.

6. How to run

cd bootloader/tools
pip install -r requirements.txt
python keygen.py

Expected output: Wrote .../bootloader/lib/boot/src/boot_pubkey.c. Then rebuild the bootloader project in STM32CubeIDE and flash with ST-Link.

7. Security notes

  • Dev fixture only — the seed stm32-bare-metal-dev-fixture is public in the repo. Treat signed images as integrity-checked for learning, not as anti-cloning in the field.
  • Production — generate a unique key pair per product line (or per device with secure element), store private key offline or in a signing server, embed only the public key in firmware. Never commit production private keys.
  • Key rotation — change KEY_SEED (or use random key generation), run keygen.py, reflash bootloader, resign all deployable images.
