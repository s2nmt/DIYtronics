This is Part 4 of the Yocto on BeagleBone Black series. It explains how this project exposes four buttons and two LEDs on the P8 header through a custom out-of-tree kernel module button-p8-keys. Unlike the standard gpio-keys input driver, this module provides a simple sysfs interface under /sys/kernel/button_p8_keys/ — easy to read from shell scripts or userspace without the Linux input subsystem.

1. Overview

The button-p8-keys package is a custom kernel module built as a separate Yocto recipe in meta-bbb. It is not part of the main kernel tree (linux-bb/). Instead, it is compiled out-of-tree against your kernel headers and loaded at boot.

FeatureDetail
Buttons4 switches — read-only via sysfs
LEDs2 GPIO outputs — read/write via sysfs
Interface/sys/kernel/button_p8_keys/
Driver typePlatform driver (bbb,button-p8-keys)
Recipemeta-bbb/recipes-kernel/button-p8-keys/

Why a custom driver? The standard Linux approach uses gpio-keys and the input subsystem (/dev/input/event*). This project uses a simpler sysfs-based driver instead:

  • Read button state with cat /sys/kernel/button_p8_keys/sw1
  • Control LEDs with echo 1 > /sys/kernel/button_p8_keys/led1
  • No evtest or input device handling required

2. How it works

  Buttons + LEDs (hardware)
         │
         │  GPIO pins on P8 header
         ▼
  Device Tree node (bbb,button-p8-keys)
         │
         ▼
  Kernel module: button_p8_keys.ko
         │
         ▼
  /sys/kernel/button_p8_keys/
      ├── sw1, sw2, sw3, sw4   (read: 0=pressed, 1=released)
      └── led1, led2            (read/write: 0 or 1)

Data flow

  1. Device tree declares GPIO pins and binds the bbb,button-p8-keys compatible string
  2. Module probe runs when the module loads — reads GPIOs from device tree
  3. Sysfs files are created under /sys/kernel/button_p8_keys/
  4. Userspace reads buttons and writes LED values through those files

3. Hardware connections

Buttons (input, active-low)

Buttons connect between the GPIO pin and GND. Internal pull-up is enabled in the device tree — released = 1, pressed = 0.

Sysfs nameBBB pinGPIO
sw1P8_12gpio1_12
sw2P8_14gpio0_26
sw3P8_16gpio1_14
sw4P8_18gpio2_1

LEDs (output)

Sysfs nameBBB pinGPIO
led1P8_8gpio2_3
led2P8_10gpio2_4

Write 0 to turn off, 1 to turn on.

4. Device tree configuration

File: linux-bb/arch/arm/boot/dts/am335x-bone-common.dtsi

Platform node

button_p8_keys {
    compatible = "bbb,button-p8-keys";
    pinctrl-names = "default";
    pinctrl-0 = <&button_p8_keys_pins>, <&p8_leds_pins>;
    button-gpios = <&gpio1 12 GPIO_ACTIVE_HIGH>,
                   <&gpio0 26 GPIO_ACTIVE_HIGH>,
                   <&gpio1 14 GPIO_ACTIVE_HIGH>,
                   <&gpio2 1 GPIO_ACTIVE_HIGH>;
    led-gpios = <&gpio2 3 GPIO_ACTIVE_HIGH>,
                <&gpio2 4 GPIO_ACTIVE_HIGH>;
    status = "okay";
};

        AM33XX_PADCONF(AM335X_PIN_GPMC_AD10, PIN_INPUT_PULLUP, MUX_MODE7)  /* P8_14 */
        AM33XX_PADCONF(AM335X_PIN_GPMC_AD14, PIN_INPUT_PULLUP, MUX_MODE7)  /* P8_16 */

5. The kernel module

Source: meta-bbb/recipes-kernel/button-p8-keys/files/button_p8_keys.c

Driver structure

The module is a platform driver that:

  • Matches device tree compatible bbb,button-p8-keys
  • Gets button GPIOs via gpiod_get_array(..., "button", GPIOD_IN)
  • Gets LED GPIOs via gpiod_get_array(..., "led", GPIOD_OUT_LOW)
  • Creates a sysfs kobject at /sys/kernel/button_p8_keys/

Sysfs interface

FileModeDescription
sw1 … sw4read-only (0444)Button state: 1 = released, 0 = pressed
led1, led2read/write (0644)LED state: write 0 or 1

From the driver source:

/* pull-up, button to GND: released=1, pressed=0 */
return sysfs_emit(buf, "%d\n", gpiod_get_value(key->gpio));

Device tree binding

static const struct of_device_id button_p8_keys_of_match[] = {
    { .compatible = "bbb,button-p8-keys" },
    {}
};

The button-gpios and led-gpios properties map to the "button" and "led" gpio specifier names used in gpiod_get_array().

Makefile

File: meta-bbb/recipes-kernel/button-p8-keys/files/Makefile

obj-m := button_p8_keys.o

Standard out-of-tree kernel module build against KERNEL_SRC provided by Yocto's module class.

6. The Yocto recipe

File: meta-bbb/recipes-kernel/button-p8-keys/button-p8-keys_1.0.bb

SUMMARY = "Kernel module: P8 buttons (input) and LEDs (output) via sysfs"
LICENSE = "GPL-2.0-only"

inherit module

DEPENDS += "virtual/kernel"

SRC_URI = "file://Makefile \
           file://button_p8_keys.c \
          "

S = "${WORKDIR}"

KERNEL_MODULE_AUTOLOAD += "button_p8_keys"
RPROVIDES:${PN} += "kernel-module-button_p8_keys"
SettingPurpose
inherit moduleYocto class for out-of-tree kernel modules

This module is independent of linux-bb.org_%.bbappend. It builds separately but requires the kernel (for headers and module ABI).

7. Adding to the image

File: meta-bbb/recipes-core/image/core-image-minimal.bbappend

IMAGE_INSTALL:append = " button-p8-keys ..."

This installs button_p8_keys.ko into the rootfs under /lib/modules/<kernel-version>/ and configures module autoload.

Without this line, the module is built in tmp/deploy/ but not present on the board.

9. Using on the board

After boot, verify the module loaded:

lsmod | grep button_p8_keys

Check sysfs:

ls /sys/kernel/button_p8_keys/
# sw1  sw2  sw3  sw4  led1  led2

Read a button

cat /sys/kernel/button_p8_keys/sw1
# 1 = not pressed
# 0 = pressed

Control an LED

echo 1 > /sys/kernel/button_p8_keys/led1   # turn on
echo 0 > /sys/kernel/button_p8_keys/led1   # turn off
cat /sys/kernel/button_p8_keys/led1        # read state

Poll buttons in a loop

while true; do
    echo -n "sw1=$(cat /sys/kernel/button_p8_keys/sw1) "
    echo -n "sw2=$(cat /sys/kernel/button_p8_keys/sw2) "
    echo -n "sw3=$(cat /sys/kernel/button_p8_keys/sw3) "
    echo "sw4=$(cat /sys/kernel/button_p8_keys/sw4)"
    sleep 0.2
done

Kernel log on probe

dmesg | grep button-p8-keys

Expected output:

buttons (input): /sys/kernel/button_p8_keys/{sw1..sw4} (1=released, 0=pressed)
leds (output): /sys/kernel/button_p8_keys/{led1,led2} = P8_8,P8_10 (write 0/1)
